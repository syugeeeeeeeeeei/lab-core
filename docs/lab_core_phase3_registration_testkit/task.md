# Task List: Phase 3 登録テスト導線整備

## 目的
アプリ登録のユーザーテストを、手作業でも再現できる形で実施しやすくする。

## 実施タスク
- 登録フォームに「テスト値入力」導線を追加
- テスト値（OruCa想定 / シンプルWeb / Headless API）を定義
- 同名衝突を避けるため、テスト値適用時に時刻サフィックスを付与
- バックエンドに fixture 取得 API を追加
- 一括登録用スクリプトを追加
- 削除確認 UI（削除モード + 確認文字列）を追加
- 説明書へユーザーテスト手順を追記

## 完了条件
- ダッシュボード上でテスト値をワンクリック入力できる
- 同じテストを繰り返しても重複エラーになりにくい
- スクリプトで3種類の登録テストをまとめて流せる
- 削除操作に二重確認がある
- 説明書に操作手順と確認観点が記載されている
