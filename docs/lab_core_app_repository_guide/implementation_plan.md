# Implementation Plan: 適合アプリリポジトリ指南書

## 方針
- 「理想論」ではなく Lab-Core 現行実装で実際に必要な条件を書く
- 仕様説明だけでなく、開発者がそのまま使えるテンプレートを載せる
- 受け入れ判定をチェックリスト化し、運用を標準化する

## 実装ステップ
1. 受け入れ仕様と API 入力要件を確認
2. compose と登録値の対応関係を整理
3. Standard/Headless の最小テンプレートを作成
4. データ永続化とデバイス要件の記述を作成
5. 自己チェック手順と失敗パターンを作成
6. OruCa 想定の補足を追記
7. docs 導線にリンクを追加

## 品質観点
- 登録時に必要な項目漏れがないか
- 実行モード差分（dry-run/execute）の誤解を招かないか
- 初学者が 1 回で compose を合わせられる内容か
