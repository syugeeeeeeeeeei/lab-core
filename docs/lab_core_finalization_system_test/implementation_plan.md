# Implementation Plan: 最終仕上げと回帰テスト

## 方針
- 手作業確認だけに依存せず、API横断の自動テストを1本化する
- 既存の dry-run モードを活かし、CI なしでも手元で再現可能にする
- 失敗時に原因を追いやすいよう、検証順序を固定する

## 実装ステップ
1. スモークテスト本体（Node）を追加
2. バックエンド起動ラッパー（Shell）を追加
3. root `package.json` に `test:smoke` を追加
4. build + smoke 実行
5. README / 説明書更新

## テスト観点
- Health / System Status
- Fixture 取得 / アプリ登録
- 再起動 / 再ビルド / 更新確認 / 更新適用 / ロールバック
- ログ取得 / インフラ同期 / 削除
- 期待失敗系（ロールバック不可時の 400）
