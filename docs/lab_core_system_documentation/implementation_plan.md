# Implementation Plan: システムドキュメント整備

## 方針
- 理想像ではなく、現行コードの実装事実に合わせて記述する
- 利用者目線（操作手順）と開発者目線（仕様/API）を分離して読みやすくする
- 既存資料と重複する場合は、参照元を明確にして更新しやすくする

## 実装ステップ
1. backend/dashboard と既存 docs を棚卸し
2. ドキュメント構成を定義（要件/仕様/機能/使い方/トラブル）
3. 各文書を実装準拠で執筆
4. 入口となる index を作成
5. README と既存説明書に導線を追加
6. 誤記・用語統一を確認

## 品質観点
- API 名、状態名、ジョブ名がコードと一致しているか
- `dry-run` と `execute` の差異が明確か
- 破壊的操作の注意点が明確か
- 初学者が最短で起動・登録・復旧まで辿れるか
