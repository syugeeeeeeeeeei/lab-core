# Task List: システムドキュメント整理

## 目的
現在の実装を基準に、要件・仕様・機能・使い方・トラブル対応を一本化した運用ドキュメントを整備する。

## 実施タスク
- ドキュメントフォルダ `docs/lab_core_system_documentation/` を新設
- 要件とスコープを整理
- 実装準拠のシステム仕様を整理
- 機能カタログを整理
- 利用者向け使い方ガイドを整理
- トラブルシューティングを整理
- README / 既存説明書から辿れる導線を追加

## 完了条件
- 利用者が「何を使えばよいか」を迷わない
- 開発者が「現在実装されている仕様」を確認できる
- 障害時に最初の切り分けを文書だけで進められる
