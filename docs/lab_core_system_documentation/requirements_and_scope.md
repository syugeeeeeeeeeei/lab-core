# 要件とスコープ

最終更新: 2026-03-18

## 1. 本システムの目的
Lab-Core v3 は、研究室内 Web アプリ運用を「詳しい人だけができる作業」から「引き継ぎ可能な日常作業」へ変えるための内部基盤です。

主目的は次の 4 点です。
- 怖くない操作
- 壊しにくい運用
- 壊れても戻しやすい復旧導線
- 引き継ぎしやすい構成

## 2. 想定利用者
- 一般利用者:
アプリ登録、再起動、ログ確認、削除を UI 中心で実施する人
- 管理利用者:
再構築、移行、設定更新、同期確認を担当する人
- 開発者:
基盤機能（API/UI/ジョブ）の改善と保守を担当する人

## 3. 管理対象
- Git リポジトリから取得できる HTTP アプリ
- アプリ単位の配備情報と公開ルート
- アプリごとのイベント、ジョブ、ログ確認導線

## 4. 管理対象外（現時点）
- Kubernetes 相当の汎用オーケストレーション
- 高度な監査ログや権限制御
- CPU/メモリの詳細メトリクス可視化
- アプリ固有データの意味を理解した論理バックアップ

## 5. 運用原則
- Fearless Operation:
初心者でも押せる UI と日本語メッセージを優先
- Simple Recovery:
再起動 → ログ確認 → 再ビルド → 再構築 の導線を固定
- Responsibility Separation:
アプリ機能はアプリ側、配備と公開は基盤側
- Internal First:
研究室 LAN 向け前提の設計

## 6. 現行実装の必須前提
- Backend: Node.js 20+ / TypeScript / Hono / SQLite
- Dashboard: React + Vite
- 実行モード:
`dry-run`（安全に導線確認）と `execute`（実際に Docker/Git 実行）
- ルートドメインと IP は `.env` で切替

## 7. 初期運用で重視している品質
- 操作前後の状態が UI で見えること
- 破壊操作に確認導線があること
- 失敗時に次に見るべき場所（イベント/ログ）が分かること
- `.env` を直接編集せずに設定できること（`yarn config:init`, `yarn config:reset`）
